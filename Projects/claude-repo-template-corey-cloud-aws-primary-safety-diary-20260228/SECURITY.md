# Security Policy

## Supported Versions

This template is a starting point. You own production hardening for your environment.

## Reporting a Vulnerability

Do not open public issues for security-sensitive reports.
Instead, contact the repository owners listed in `CODEOWNERS` (or your internal security team).

## Secure Defaults Included

- secret scanning workflow (`gitleaks`)
- dependency vulnerability scan (`osv-scanner`)
- SAST baseline (`semgrep`)
- SBOM generation (CycloneDX)
