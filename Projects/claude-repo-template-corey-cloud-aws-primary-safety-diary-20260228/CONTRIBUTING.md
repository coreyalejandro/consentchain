# Contributing

## One correct path

1. Create a branch.
2. Keep changes small and atomic.
3. Run `./scripts/verify.sh`
4. Run security checks: `./scripts/security.sh`
5. Open a PR with:
   - what changed
   - verification output
   - risk level
   - rollback plan if risk is MED/HIGH

## Commit discipline

- Prefer conventional commits when possible.
- No "WIP" merges to main.
