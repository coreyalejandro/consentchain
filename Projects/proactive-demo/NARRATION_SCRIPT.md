# PROACTIVE Demo — Voiceover Narration Script

**Total runtime:** ~4 minutes 12 seconds
**Tone:** Confident, conversational, like explaining your work to a smart colleague
**Pacing:** Natural pauses between sections. No rush.

---

## [0:00–0:05] Intro Card

> This is PROACTIVE — a constitutional AI safety agent built for the software development lifecycle.

## [0:05–0:50] Scene: Repository Context

> Let me walk you through a live demonstration.

> Here we have a GitLab merge request. A developer has submitted what they claim is a complete feature implementation.

> At first glance, the code structure looks reasonable. There are files, functions, and even some documentation.

> But appearances can be deceiving — and that's exactly the kind of problem PROACTIVE was designed to catch.

## [0:50–1:45] Scene: Opening the Merge Request

> Let's look more closely. The commit message says "full implementation" — confident language.

> But when we inspect the actual code, several key functions are empty stubs. They have the right signatures, the right names, but no real logic inside.

> This is what we call phantom completion — the code presents a facade of completeness without delivering on its promises.

> A traditional code review might miss this. The structure looks right. But the substance is hollow.

## [1:45–2:20] Scene: Agent Begins Review

> Now watch what happens when PROACTIVE enters the picture.

> The agent doesn't just scan for syntax errors or style violations. It reasons about intent versus implementation.

> It applies constitutional principles — rules derived from real-world contract failures and AI safety research — to evaluate whether the code actually does what it claims.

## [2:20–3:20] Scene: PROACTIVE Analysis

> Here's the analysis in real time. PROACTIVE has identified an invariant violation.

> Specifically, it's flagged phantom completion — that's violation class I2 in our constitutional framework.

> The agent traces evidence through the code paths, connecting the confident claims in the commit message to the empty function bodies in the actual implementation.

> This isn't a simple linter finding. This is structured reasoning about the gap between what was promised and what was delivered.

## [3:20–4:07] Scene: Violations & Enforcement

> The violation report surfaces another finding — false confident claims, class F1.

> The developer's language implied certainty and completeness. The code tells a different story. PROACTIVE catches that dissonance.

> And here's the enforcement. Based on the constitutional violations detected, PROACTIVE blocks the merge. The code cannot proceed until the gaps are addressed.

> This is safety-first development — catching integrity failures before they reach production.

## [4:07–4:12] Closing Card

> PROACTIVE. Constitutional AI safety for the software development lifecycle.
