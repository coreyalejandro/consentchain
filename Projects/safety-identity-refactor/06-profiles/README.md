# 06-profiles — Distribution

## What This Is

Profiles are the distribution layer of Safety Systems Design.

**Rule:** Profiles = distribution. Profiles communicate this work to specific audiences in the format those audiences require.

A profile is not a resume. It is a curated view of the identity, platform, and products — filtered for audience, context, and channel.

## Profile Types

**Audience:** Hiring managers, engineering leads, technical collaborators
**Format:** GitHub, LinkedIn (technical), portfolio `/documentation` route
**Emphasis:** Implementation evidence — repos, test coverage, invariants, architecture decisions

Key signals for this audience:

### Research Profile

**Audience:** AI safety researchers, fellows programs (Anthropic, etc.)
**Format:** Research papers, application submissions, safety conference contexts
**Emphasis:** COL-PROACTIVE thesis, ITAYN framework, epistemic reliability as safety requirement

Key signals for this audience:
- The safety axiom: epistemic reliability is a safety requirement, not a quality feature
- The 35-year arc: educator → dean → AI engineer as lived evidence of the doctrine
- COL-PROACTIVE: MBSE bridge between "Attention Is All You Need" and "Constitutional AI"
- The four-domain taxonomy: Epistemic Safety, Human Safety, Cognitive Safety, Empirical Safety
- Empirical Safety Engine (ESE): measurement layer that proves safety rather than claiming it

### Creative/Design Profile

**Audience:** Design-forward companies, creative technologists, innovation teams
**Format:** Portfolio visual surface, Creative Chaos design system docs
**Emphasis:** Creative Chaos, interactive demos, animation + 3D effects

Key signals for this audience:

### Accessibility-First Profile

**Audience:** Organizations prioritizing inclusive design, accessibility advocates
**Format:** UICare positioning, COL thesis applied to UI
**Emphasis:** Dis-enabled frame, cognitive load reduction, safety as precondition for usability
